## ✅ Финальные архитектурные решения

### **1. Векторный подход вместо ручных признаков**
```
❌ Отменено: ручное извлечение признаков (has_person, background_type, text_coverage)
✅ Утверждено: CLIP embedding как универсальный семантический код изображения
```

---

### **2. Метрики эффективности: только косвенные, но валидируемые**
| Метрика | Статус | Как используется |
|---------|--------|-----------------|
| Позиция в поиске | ✅ | Целевая переменная для корреляции |
| Скорость отзывов/день | ✅ | Прокси для продаж/трафика |
| Динамика рейтинга | ✅ | Индикатор удовлетворённости |
| Наличие бейджей | ✅ | Бинарный маркер успеха площадки |
| Полнота карточки | ✅ | Контрольная переменная в анализе |

> ⚠️ Все метрики интерпретируются как **корреляции**, не причинно-следственные связи.

---

### **3. Статистическая валидация паттернов (обязательный слой)**
```python
# Утверждённый минимальный набор проверок:

1. Пермутационный тест → p-value < 0.05
2. Бутстрэп ЦИ центроида → узкий интервал = устойчивость
3. Эффект размера → практическая значимость (не менее +3-5 позиций)
4. Temporal holdout → воспроизводимость на новых данных
```

**Формат вывода инсайта:**
```
«Паттерн кластера #2 значим (p=0.02), 
карточки, близкие к центроиду, в среднем на +7.3 позиции выше 
(95% CI: 2.1–12.4), эффект воспроизведён на тестовом периоде»
```

## 🔬 Исследовательский протокол (утверждённый)

### **Этап 1: Микро-исследование (недели 1-2)**
```
1. Категория: одна узкая (например, «Чехлы для iPhone»)
2. Выборка: топ-20 карточек по позиции + 20 из хвоста (контроль)
3. Обработка: CLIP embedding для всех изображений
4. Анализ:
   - Кластеризация топ-20 (HDBSCAN, min_cluster_size=3)
   - Расчёт центроидов кластеров
   - Пермутационный тест: «случайны ли эти кластеры?»
5. Вывод: 1-2 гипотезы в формате «вектор → позиция»
```

### **Этап 2: Микро-генерация и тест (недели 3-4)**
```
1. Товар: один свой (или тестовый)
2. Генерация:
   - Вариант А: базовый (контроль)
   - Вариант Б: guided by successful centroid (CLIP-guided diffusion)
3. Тест:
   - Если площадка позволяет: A/B запуск
   - Если нет: слепая оценка 5-10 людьми (предпочтение)
4. Метрика успеха: изменение прокси-метрики за 7-14 дней
```

## 🎯 Критерии успеха системы (утверждённые)

```
✅ Инсайт считается валидным, если:
   - Выборка ≥ 30 карточек в группе
   - p-value (permutation) < 0.05
   - Эффект размера ≥ +3 позиции (или эквивалент в конверсии)
   - Воспроизводимость на temporal holdout

✅ Система считается рабочей, если:
   - Время от парсинга до инсайта ≤ 24 часа
   - Точность кластеризации (silhouette) > 0.4 для топ-сегмента
   - Генерация варианта под паттерн ≤ 5 минут на карточку

✅ Ценность для пользователя:
   - «Я не гадаю, что работает — я вижу паттерны, подтверждённые данными»
   - «Я тестирую гипотезы за дни, а не за месяцы»
   - «Система учится на результатах моих запусков»
```

## 🔄 Обратная связь и эволюция системы

```
Мониторинг дрейфа:
├─ Еженедельно: пересчёт центроидов топ-кластеров
├─ Если центроид сдвинулся > порога → алерт «тренд изменился»
├─ Автоматическая перекалибровка генератора под новые паттерны

Обучение на своих результатах:
├─ После запуска своей карточки → отслеживание её позиции/отзывов
├─ Если карточка «выстрелила» → её вектор добавляется в пул успешных
├─ Следующая генерация учитывает обновлённые паттерны
```

---

> **Философия финального решения:**  
> Не «копировать успешных», а «выявлять статистически устойчивые визуальные паттерны и адаптировать их под свои задачи».  
> Векторное пространство + минимальная статистическая строгость = баланс между скоростью и надёжностью.

### **1. Image-to-Text (описание изображения)**

```python
# qwen_vl_analyzer.py
from transformers import Qwen2VLForConditionalGeneration, AutoTokenizer, AutoProcessor
from qwen_vl_utils import process_vision_info
import torch

class QwenVLAnalyzer:
    def __init__(self):
        self.model = Qwen2VLForConditionalGeneration.from_pretrained(
            "Qwen/Qwen2-VL-7B-Instruct",
            torch_dtype=torch.float16,
            device_map="auto"
        )
        self.processor = AutoProcessor.from_pretrained("Qwen/Qwen2-VL-7B-Instruct")
    
    def describe_product_image(self, image_path):
        """
        Детальное описание товарной карточки
        """
        messages = [
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "image": image_path,
                    },
                    {
                        "type": "text",
                        "text": """
                        Проанализируй изображение товарной карточки для маркетплейса.
                        
                        Опиши:
                        1. Композиция: где расположен товар (центр, слева, справа), какой процент кадра занимает
                        2. Фон: цвет, текстура, однородность
                        3. Товар: как показан (крупно/мелко, под каким углом, один или несколько)
                        4. Люди/модели: есть ли, что делают
                        5. Текст на изображении: есть ли надписи, где, какой размер
                        6. Дополнительные элементы: логотипы, иконки, бейджи, рамки
                        7. Освещение: студийное/естественное, тени, блики
                        8. Цветовая гамма: основные цвета, контраст
                        9. Стиль: минимализм, коллаж, лайфстайл, инфографика
                        
                        Верни JSON:
                        {
                          "composition": "...",
                          "background": "...",
                          "product_placement": "...",
                          "has_people": true/false,
                          "text_elements": ["...", "..."] | [],
                          "additional_elements": ["...", "..."] | [],
                          "lighting": "...",
                          "color_palette": ["...", "..."],
                          "style": "...",
                          "overall_description": "краткое описание одним абзацем"
                        }
                        """
                    }
                ]
            }
        ]
        
        text = self.processor.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=True
        )
        image_inputs, video_inputs = process_vision_info(messages)
        inputs = self.processor(
            text=[text],
            images=image_inputs,
            videos=video_inputs,
            padding=True,
            return_tensors="pt"
        )
        inputs = inputs.to("cuda")
        
        generated_ids = self.model.generate(**inputs, max_new_tokens=512)
        generated_ids_trimmed = [
            out_ids[len(in_ids):] for in_ids, out_ids in zip(inputs.input_ids, generated_ids)
        ]
        output_text = self.processor.batch_decode(
            generated_ids_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False
        )[0]
        
        import json
        return json.loads(output_text)
```
