# Cardgen — план (MVP A: «студийное качество из домашнего фото»)

## Цель

Загрузка фото товара → скачивание улучшенного изображения: свет, цвет, резкость, шум; опционально белый фон под требования маркетплейса. Без парсинга WB, кластеризации и генерации lifestyle в MVP.

**Критерий успеха:** на тестовом наборе ≥70% кадров помечаются как «можно заливать как есть».

---

## Сегмент: бытовая электроника и гаджеты (WB)

**ICP:** селлеры бытовой электроники и аксессуаров для гаджетов, главное фото с телефона → визуал «как в студии»; при необходимости **белый фон** под требования WB (см. `info/Пакеты услуг.md`).

**Почему этот сегмент:** для многих товаров в ниже перечисленных категориях на главном фото ожидается **нейтральный/белый фон**; корпуса и комплект чаще **проще для автоматического выреза**, чем одежда или сложное стекло.

**Не обещать в MVP без доработок:** сильные блики на глянце, отражающий экран «как зеркало», полупрозрачный пластик — возможны артефакты маски; фиксировать в фидбеке.

---

## Фаза 0 — Границы ✅

- **Вход:** 1–20 изображений (JPG/PNG/WebP).
- **Выход:** те же кадры, улучшенные; опция белого фона.
- **Не в MVP:** парсинг конкурентов, кластеры, шаблоны «как у топа», AR/360.

---

## Фаза 1 — Датасет и критерии (ручная работа) — фокус: электроника

- Собрать **25–40** фото **именно бытовой электроники / гаджетов** (желательно свои или с согласия): плохой свет, жёлтый баланс, шум, «грязный» фон, кадр с телефона.
- Класть исходники в **`samples/electronics/in/`** (репозиторий: только `in/`, результаты не коммитим).
- Для каждого кадра проставить **да/нет** по чеклисту «готово залить на WB» (достаточно таблицы или заметок).

**Чеклист качества (электроника, главное фото):**

| # | Критерий |
|---|----------|
| 1 | Белый фон **#FFFFFF** без заметного градиента (если использовали `--white-bg`) |
| 2 | Нет сильного жёлтого/синего каста на корпусе |
| 3 | Текст на корпусе/наклейке читаем (не мыло) |
| 4 | Край товара без явной «ореола» от маски |
| 5 | Товар узнаваем, форма не искажена |
| 6 | Нет лишнего мусора на фоне (если без выреза — фон хотя бы нейтральный) |

**Команда для прогона тестового набора:**

```bash
python -m cardgen process --in samples/electronics/in --out samples/electronics/out
python -m cardgen process --in samples/electronics/in --out samples/electronics/out_white --white-bg
```

**Опционально — подтянуть примеры карточек из поиска WB** (первое фото каталога, `.webp`; только для **локальных** тестов пайплайна, не для публикации):

```bash
python -m cardgen wb-download --query "бытовая техника" --out samples/electronics/in --max 20 --delay 0.5
```

Права на изображения у селлеров/WB; при **429 Too Many Requests** уменьшите `--max`, увеличьте `--delay`, повторите позже или с другого IP.

**Метрика:** доля кадров, где все применимые пункты чеклиста = да → цель **≥70%**.

## Фаза 2 — CLI: папка → папка ✅

- Скрипт: `in/` → `out/` с логом.
- Цепочка: апскейл (если мало пикселей) → баланс белого → контраст (CLAHE в LAB) → лёгкое шумоподавление → unsharp.
- Опция `--white-bg`: сегментация товара (`rembg`), композит на `#FFFFFF` (без отдельной тени).

**Установка (один раз):**

```powershell
cd d:\langs\python\cardgen
python -m venv .venv
.\.venv\Scripts\pip install -e .
```

**Команда:**

```bash
python -m cardgen process --in in --out out
python -m cardgen process --in in --out out --white-bg
```

Пример в репозитории: синтетика в `samples/in/`; для **ниши электроники** — исходники в **`samples/electronics/in/`**, результаты в `samples/electronics/out/` или `samples/electronics/out_white/` (не коммитим).

**Стоп-критерий:** улучшение заметно на большинстве тестовых фото без артефактов «пластика».
